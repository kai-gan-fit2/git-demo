# 🧰 Docker & JumpServer 故障排查速查手册（极简版）

## ① 第一步：看容器是否在跑

```Bash

docker ps
docker ps -a
```

- **状态 Exited** → 起不来

- **状态 Up** → 运行正常

- **端口没有** → 访问不了

## ② 第二步：看日志（90% 问题在这里）

```Bash

docker logs -f 容器名
```

重点找：

- error

- fail

- connection refused

- permission denied

- 连接不上 Core

## ③ 第三步：检查端口

```Bash

ss -tulpn | grep 端口
```

常见问题：

- 端口被占用 ❌

- 端口没映射 ❌

- 防火墙没放行 ❌

## ④ 第四步：进入容器自测

```Bash

docker exec -it 容器名 /bin/bash
```

进去后测试：

- 看挂载目录是否正常

- 看文件权限

- 测试网络连通性

- 看内部端口是否监听

## ⑤ 第五步：检查挂载与权限

```Bash

ls -l 挂载目录
```

常见问题：

- 挂载目录不存在

- 权限不足

- 目录为空导致覆盖容器文件

# ⚡️ 8 大高频故障 + 10 秒修复

## 1. 容器启动就退出

```Plain Text

docker logs 容器名
```

根据日志改正即可。

## 2. 访问不了（连不上）

先查：

```Plain Text

docker ps
```

看端口是否映射。

## 3. 端口不通

```Plain Text

ufw allow 端口
```

或云服务器放通安全组。

## 4. 挂载后空白/报错

```Plain Text

临时删掉 volumes 测试
```

正常就是挂载问题。

## 5. 权限拒绝

```Plain Text

chmod -R 755 目录
chown -R 1000:1000 目录
```

## 6. Koko 连不上 Core

检查网络是否同网：

```Plain Text

docker network ls
```

## 7. 健康检查失败

```Plain Text

docker inspect 容器名 --format '{{.State.Health.Status}}'
```

资源不够或服务未启动。

## 8. 中文乱码

```Plain Text

echo '<meta charset="UTF-8">' > 文件
```
